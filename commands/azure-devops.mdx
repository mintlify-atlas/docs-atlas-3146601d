The `yitpush azure-devops` command provides interactive tools for managing Azure DevOps resources, including creating and cloning repositories, and browsing variable groups.

## Requirements

<Warning>
  This command requires the [Azure CLI](https://docs.microsoft.com/en-us/cli/azure/install-azure-cli) to be installed and configured.
</Warning>

Before using these commands:
1. Install Azure CLI from [Microsoft's official site](https://docs.microsoft.com/en-us/cli/azure/install-azure-cli)
2. Log in to Azure: `az login`
3. The Azure DevOps extension will be installed automatically if missing

## repo new

Create a new repository in Azure DevOps with interactive prompts.

### Usage

```bash
yitpush azure-devops repo new
```

### Features

- **Auto-detection**: Automatically detects Azure DevOps organizations
- **Smart defaults**: Suggests current directory name as repository name
- **Git initialization**: Creates a git repository if one doesn't exist
- **Remote management**: Handles existing remotes (choose `azure`, `origin`, or custom name)
- **Interactive navigation**: "← Back" option at every step
- **Windows support**: Automatically detects `az.cmd` on Windows

## Platform support

<Info>
  **Windows users**: YitPush automatically detects and uses `az.cmd` on Windows systems. No additional configuration is needed.
</Info>

The command works on:
- **macOS**: Uses `az` command directly
- **Linux**: Uses `az` command directly  
- **Windows**: Automatically uses `cmd.exe /c az` to invoke `az.cmd`

## Troubleshooting

### Azure CLI not installed

```
❌ Azure CLI (az) is not installed.
Please install it from: https://docs.microsoft.com/en-us/cli/azure/install-azure-cli
```

**Solution**: Install the Azure CLI from Microsoft's official website.

### Not logged in to Azure

```
❌ Please log in to Azure CLI first.
Run: az login
```

**Solution**: Run `az login` and complete the authentication flow.

### No organizations found

```
⚠️  No organizations found.
Please enter your Azure DevOps organization name manually:
>
```

**Solution**: If auto-detection fails, you can manually enter your organization name (e.g., `MyCompany` for `https://dev.azure.com/MyCompany`).

### No repositories found

```
❌ No repositories found in project 'WebApp'.
```

**Solution**: Verify the selected project contains repositories or has the correct permissions.

### No variable groups found

```
❌ No variable groups found in project 'WebApp'.
```

**Solution**: Verify the project has variable groups configured or check your permissions.
