title: 'yitpush pr'
description: 'Generate AI-powered pull request descriptions by comparing branches'
---

The `yitpush pr` command generates comprehensive pull request descriptions by analyzing the differences between two branches using DeepSeek AI.

## Basic usage

```bash
yitpush pr
```

The command will interactively prompt you to:
1. Select the **source branch** (the branch with your changes)
2. Select the **target branch** (the branch you want to merge into)
3. Generate an AI-powered PR description based on the diff

## Flags

<ParamField path="--detailed" type="boolean" default="false">
  Generate a detailed PR description with multiple sections including summary, detailed changes, files changed, testing suggestions, and notes about breaking changes.
</ParamField>

<ParamField path="--language" type="string" default="english">
  Specify the output language for the PR description. Accepts any language name (e.g., `spanish`, `french`, `japanese`). Can be shortened to `--lang`.
</ParamField>

<ParamField path="--save" type="boolean" default="false">
  Save the generated PR description to a markdown file. The file is named `pr-description-<source>-to-<target>.md`.
</ParamField>

## How it works

### Step 1: Fetch branches

Retrieves all local and remote branches, sorted by last commit date.

### Step 2: Select source branch

Prompts you to select the branch that contains your changes (typically your feature branch).

### Step 3: Select target branch

Prompts you to select the branch you want to merge into (typically `main`, `develop`, or `master`).

### Step 4: Generate diff

Compares the branches using:
```bash
git diff <target-branch>...<source-branch>
```

The `...` syntax (three dots) compares the tip of the source branch with the merge-base of the two branches, showing only the changes introduced in the source branch.

### Step 5: AI analysis

Sends the diff to DeepSeek AI with context about:
- Commit message style (conventional commits)
- Markdown formatting requirements
- The specified detail level
- The target language

### Step 6: Output

Displays the generated description in a formatted panel and:
- Copies it to your clipboard
- Optionally saves it to a file

## PR description format

### Basic mode

Includes:
- **Title**: Conventional commit style (e.g., `feat: add user authentication`)
- **Summary**: Brief paragraph explaining the purpose
- **Changes**: Bullet list of key changes

### Detailed mode

Includes:
- **Title**: Conventional commit style
- **Summary**: Comprehensive explanation of the changes
- **Changes**: Detailed bullet list grouped by category (features, fixes, refactoring, etc.)
- **Files Changed**: List of key files with descriptions of what changed
- **Testing**: Suggestions for testing the changes
- **Notes**: Breaking changes, migration steps, or other important information

<Info>
  All PR descriptions are generated in Markdown format, ready to paste into GitHub, GitLab, Azure DevOps, or any other platform.
</Info>

## Navigation

<Note>
  You can go back at any step by selecting "← Back" from the menu.
</Note>

If you select "← Back" during:
- **Source branch selection**: Exits the command
- **Target branch selection**: Returns to source branch selection

## Requirements

- Must be run from within a git repository
- `DEEPSEEK_API_KEY` environment variable must be set
- Internet connection required to call the DeepSeek API
- At least 2 branches must exist in the repository

## Troubleshooting

**No differences found**
```
⚠️  No differences found between the selected branches.
```
This means the two branches are identical. Make sure you've committed changes to your source branch.

**Need at least 2 branches**
```
❌ Need at least 2 branches to compare.
```
Create at least one additional branch to use this command.

<Warning>
  Make sure you have set the `DEEPSEEK_API_KEY` environment variable before using this command. See [Configuration](/guides/configuration) for setup instructions.
</Warning>
