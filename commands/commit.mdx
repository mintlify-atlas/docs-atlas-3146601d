title: 'yitpush commit'
description: 'Stage, commit and push changes with an AI-generated message'
---

The `yitpush commit` command analyzes your git changes, generates a conventional commit message using DeepSeek AI, and automatically pushes to your remote repository.

## Basic usage

```bash
yitpush commit
```

This command performs the following steps:
1. Analyzes all git changes (staged, unstaged, and untracked files)
2. Generates a conventional commit message using DeepSeek Reasoning AI
3. Stages all changes with `git add .`
4. Commits with the generated message
5. Pushes to the remote repository

## Flags

<ParamField path="--confirm" type="boolean" default="false">
  Ask for confirmation before committing and pushing. When enabled, you can review the generated message and approve or cancel the operation.
</ParamField>

<ParamField path="--detailed" type="boolean" default="false">
  Generate a detailed commit message with title and body. The body includes descriptive paragraphs, bullet points for key changes, and mentions important files modified.
</ParamField>

<ParamField path="--language" type="string" default="english">
  Specify the output language for the commit message. Accepts any language name (e.g., `spanish`, `french`, `japanese`). Can be shortened to `--lang`.
</ParamField>

<ParamField path="--save" type="boolean" default="false">
  Save the generated commit message to a markdown file. The file is named `commit-message-YYYYMMDD-HHMMSS.md`.
</ParamField>

## Workflow details

### Change detection

The command analyzes three types of changes:
- **Staged changes**: Already added with `git add`
- **Unstaged changes**: Modified tracked files
- **Untracked files**: New files not yet tracked by git

All changes are combined into a single diff for AI analysis.

### No changes behavior

If there are no changes to commit, the command will attempt to push existing commits:

```bash
⚠️  No changes detected in the repository.

Will attempt to push existing commits...

⏩ No changes to commit, proceeding with push...
```

### Upstream branch handling

If your current branch has no upstream remote:

- **Without `--confirm`**: Automatically sets upstream to `origin/<branch-name>`
- **With `--confirm`**: Prompts you to confirm setting the upstream

<Note>
  The commit message is automatically copied to your clipboard for easy reuse.
</Note>

<Warning>
  Make sure you have set the `DEEPSEEK_API_KEY` environment variable before using this command. See [Configuration](/guides/configuration) for setup instructions.
</Warning>

## Requirements

- Must be run from within a git repository
- `DEEPSEEK_API_KEY` environment variable must be set
- Internet connection required to call the DeepSeek API
- Git must be installed and configured
